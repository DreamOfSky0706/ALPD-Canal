# SJ · 09——视检

一个基于 OpenCV 的根管口检测程序。

---

## 算法概述

最终程序并非题目描述的标准四步，原因在后面的 SVM 使用情况说明里会详细解释，简单来说，在我的能力范围内，SVM 模型的区分能力不足以独立完成检测，所以我在 SVM 之外手搓了评分系统，SVM 起辅助作用，进行了一个面向结果编程的操作。

完整的处理流程：

```
原始图像
    │
    ├── 牙齿掩膜提取
    │     ├── 路径A：颜色特征分割（R-B差 / 亮度 / R-G差）
    │     └── 路径B
    │           ├── LAB 多通道多尺度梯度边界分割
    │           ├── 中心加权颜色分割
    │           ├── GrabCut
    │           └── Otsu 兜底
    │
    ├── 掩膜后处理
    │     ├── 小掩膜扩展
    │     ├── 凸包收紧 + 亮度边界清理
    │     └── 腐蚀
    │
    ├── ROI 提取 + CLAHE 对比度增强
    │
    ├── 候选区域生成
    │     ├── 暗区检测
    │     ├── 暗度补充
    │     └── 暗点定位
    │
    ├── 候选过滤
    │
    ├── 综合评分
    │     ├── 物理特征评分（对比度 / 深度 / 圆度 / 长宽比 / 亮度 / 中心距离）
    │     └── SVM 辅助
    │
    ├── 无候选点时暴力搜索
    │
    ├── 后处理
    │     ├── NMS
    │     ├── ROI 边缘过滤
    │     ├── 牙齿掩膜过滤
    │     ├── 亮度过滤
    │     ├── 聚类合并
    │     └── TopK 保留
    │
    └── 诊断报告生成
```

## 已知问题与不足

### 检测准确率

测试结果：

| 指标 | 数值         |
|------|------------|
| 正确检测的图像 | 15 (62.5%) |
| 根管口总数 | 51         |
| 正确检测 | 41 (80.4%) |
| 错检 | 2          |
| 漏检 | 8          |


### 漏检的主要原因

1. 聚类合并把两个相邻根管口合成了一个。
2. 有些根管口很浅，和周围牙齿表面的亮度差异很小，没有被选为候选。
3. 个别图像的牙齿掩膜覆盖范围过小，真正的根管口落在了掩膜边缘或外部。

### 参数

评分函数的所有阈值和权重都是手动调的，不是通过网格搜索得到的。

## SVM 使用情况说明

### 发生了什么

题目提供的 `128_svm_model.xml` 期望的特征维度是 54756。为了匹配这个维度，HOG 参数必须设为：

```
window = 160×160, block = 8×8, stride = 4×4, cell = 4×4, bins = 9
```

这和 `window=32, block=16, cell=8, stride=8` 不一致。

维度匹配上之后，我以为 SVM 能正常工作了，可实际跑起来发现 SVM 对所有候选窗口的决策分数都是正值，范围大约在 0.2 到 1.6 之间。没有一个候选被挑出。

### 我做了什么

- 把 SVM 分数打印出来做诊断。
- 在确认是根管口的位置和确认不是根管口的位置分别采样，比较分数。
- 发现 SVM 确实在区分，根管口附近的分数（0.2~0.4）低于非根管口区域（0.7~1.6），但决策边界不在零。
- 尝试过纯 SVM 滑窗检测（遍历 ROI 的每个位置，提取 patch，算 HOG，送 SVM），效果很差，错检和漏检都有 1mol。
- 最终决定 SVM 做辅助评分。SVM 分数低的候选加权，分数高的候选惩罚，但不用 SVM 做唯一标准。

### 为什么不用纯 SVM

SVM 分数范围太窄，根管口候选和非根管口候选的分数存在重叠，区分能力不足以独立做检测。

### 反思

我怀疑问题出在训练时的窗口处理方式和我的不同。我的 patch 裁剪方式可能和训练时不一致，导致特征分布偏移。也有可能是我速成的 SVM 相关知识本身掌握不牢，没能正确地充分发挥其用途。
